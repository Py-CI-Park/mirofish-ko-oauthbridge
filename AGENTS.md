# AGENTS.md

이 문서는 저장소 루트 이하 전체에 적용됩니다.

## 기본 작업 원칙
- 사용자가 명시적으로 요청한 작업을 우선합니다.
- 설정/문서/코드가 함께 엮인 변경은 가능한 한 한 번에 정리하되, 검증 가능한 범위로 유지합니다.
- 완료 주장 전에는 실제 명령 실행 결과를 확인합니다.

## 커밋 작성 규칙
사용자가 커밋을 요청했을 때 에이전트는 아래 형식을 기본으로 사용합니다.

### 커밋 메시지 언어
- 커밋 제목과 본문은 **한국어**로 작성합니다.
- 본문은 **마크다운 형식**을 사용합니다.
- 첫 줄은 마크다운 헤더가 아닌 **한 줄짜리 의도(intent) 문장**으로 작성합니다.
  - 무엇을 바꿨는지보다 **왜 바꾸는지**를 우선 설명합니다.

### 커밋 본문 권장 구조
```text
<의도 한 줄>

## 배경
- 변경이 필요했던 이유
- 사용자 요청 또는 운영상 제약

## 변경 내용
- 핵심 수정 사항
- 설정 / 코드 / 문서 반영 사항

## 검증
- 실행한 확인 명령
- 확인된 결과

Constraint: <외부 제약>
Rejected: <고려했지만 채택하지 않은 대안> | <기각 이유>
Confidence: <low|medium|high>
Scope-risk: <narrow|moderate|broad>
Directive: <미래 작업자를 위한 주의사항>
Tested: <실제로 검증한 내용>
Not-tested: <아직 검증하지 못한 내용>
```

### Lore trailer 규칙
- 저장소의 Lore Commit Protocol을 유지하기 위해 trailer 키는 아래 **영문 키**를 사용합니다.
- 값은 한국어로 작성해도 됩니다.
- 최소 권장 항목:
  - `Constraint:`
  - `Confidence:`
  - `Scope-risk:`
  - `Tested:`
  - `Not-tested:`
- 대안을 검토했다면 `Rejected:`를 반드시 남깁니다.
- 후속 작업자 주의가 필요하면 `Directive:`를 남깁니다.

## 커밋 전 체크
- 변경 파일 목록을 확인합니다.
- 설정 기본값을 바꿨다면 실제 런타임에서 읽히는 값을 확인합니다.
- CLI/서버/스크립트 연동 변경은 가능한 범위에서 스모크 테스트를 수행합니다.

## 사용자 안내 규칙
- 서버 재시작이 필요한 변경이라면 최종 응답에 **재시작 필요 여부와 순서**를 반드시 적습니다.
- 아직 하지 않은 검증은 완료한 것처럼 말하지 않습니다.
