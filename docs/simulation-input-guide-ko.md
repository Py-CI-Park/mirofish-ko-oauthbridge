# 시뮬레이션 입력 가이드

Step2에서 좋은 결과를 얻으려면 **좋은 첨부파일 + 좋은 시뮬레이션 프롬프트**를 함께 써야 합니다.

- 첨부파일은 누가, 무엇을, 언제, 왜를 보여 주는 seed여야 합니다.
- 프롬프트는 어떤 대상이, 어떤 기간에, 어떤 반응을 보이는지 좁혀 줘야 합니다.
- 기술 문서, 지표 목록, 변수 목록만 있는 파일은 Step2에서 실패할 수 있습니다.
- 준비가 잘 안 되면 `필터 완화 재시도`를 검토해 더 넓게 다시 읽을 수 있습니다.

## 좋은 첨부파일 예시

- 사건 공지문, 정책 안내문, 공시문처럼 현실 단서가 분명한 문서
- 커뮤니티 반응이 들어 있는 게시글 묶음, 댓글 스레드, 캡처 요약
- 인물, 조직, 커뮤니티, 역할, 관계가 함께 적힌 메모
- 시간 흐름과 배경 맥락이 같이 있는 시나리오 노트

이런 파일은 Step2가 엔티티와 관계를 읽어 시뮬레이션용 구조를 만들기 쉽습니다.

## 부적합한 입력 예시

- 표와 수치만 있는 기술 문서
- 변수명, 파라미터명, 설정값만 나열된 파일
- 사람, 조직, 사건 설명이 없는 로그 덤프
- 맥락 없이 PDF 몇 개만 묶은 자료

이런 입력은 시뮬레이션 대상이 무엇인지 드러나지 않아서, 시뮬레이션용 엔티티가 부족해질 수 있습니다.

## 좋은 시뮬레이션 프롬프트 예시

```text
이 공지에 대한 개인 투자자, 기관 계정, 분석가, 커뮤니티 운영자, 기자 계정의 반응을
공식 발표 직후 24시간 기준으로 시뮬레이션해 주세요.
특히 어떤 문장이 갈등을 키우는지, 어떤 계정이 여론을 증폭하는지 분리해서 보여 주세요.
```

```text
업로드한 seed 문서를 바탕으로, 한국 사용자 커뮤니티와 운영자 커뮤니티가 어떤 쟁점을 두고 충돌하는지
D+0부터 D+3까지 시간대별로 시뮬레이션해 주세요.
반응 강도, 주요 주장, 반박, 확산 경로를 함께 정리해 주세요.
```

## 나쁜 프롬프트 예시

```text
분석해줘
이 자료로 시뮬레이션해줘
반응이 어떻게 될지 알려줘
좋은 결과 나오게 처리해줘
```

이런 프롬프트는 대상, 기간, 관찰 기준이 너무 넓거나 비어 있어서 결과 품질이 쉽게 흔들립니다.

## 왜 이런 차이가 결과 품질에 영향을 주나

- Step2는 첨부파일에서 행위자, 관계, 사건 흐름, 발화 단서를 읽어 시뮬레이션용 엔티티를 만듭니다.
- 좋은 첨부파일은 누가 말하는지가 보이고, 좋은 프롬프트는 언제, 누구 기준으로, 무엇을 볼지 정합니다.
- 반대로 기술 문서나 변수표만 있으면 사회적 맥락이 부족해지고, 프롬프트가 모호하면 결과가 넓고 흐려집니다.

## Step2에서 바로 쓰는 기준

- 좋은 입력은 사람, 조직, 커뮤니티, 사건, 시간, 관계가 함께 보입니다.
- 첫 결과가 비어 있거나 너무 적으면 `좋은 입력` 예시를 다시 확인합니다.
- 그래도 준비가 안 되면 `필터 완화 재시도`를 선택해 더 넓은 범위로 다시 읽습니다.
